## What's changed

A fixes-and-ergonomics release. The native layer is untouched — the same Ant Media Android
`v2.17.2`, the same iOS commit [`47e2b02`][ios-commit], the same libwebrtc `150.0.0` on
Mac Catalyst — everything here is in the managed layer, the packaging, and the supply chain
around the natives.

### Client fixes

- **A failed start now releases the camera.** The cleanup path stopped the native session
  *after* forgetting which stream it was — a no-op on Apple, a stop of stream `""` on Android —
  so a timeout, a cancellation or a server error during `PublishAsync` left the camera, the
  microphone and the peer connection live (and the iOS camera indicator on). Cleanup now stops
  first and forgets second, which is what the documentation always claimed.
- **`DataReceived` now fires on Android.** The data-channel observer was never registered with
  the SDK's builder, so receiving silently did not work there — sending did, which made it easy
  to ship a chat feature tested on iOS that was broken on Android. Enabled whenever
  `DataChannelEnabled` is set.
- **`Stop()` and `Dispose()` release a caller awaiting `PublishAsync`/`PlayAsync`** —
  `Stop()` as an `OperationCanceledException`, `Dispose()` as an `ObjectDisposedException` —
  instead of leaving the await to run out the full timeout and then throw a misleading
  "server did not confirm".
- **Known failures fail fast.** Playing a stream that does not exist, or publishing with a taken
  id, stalled for the whole 30-second timeout because the Android SDK reports those through
  dedicated callbacks the bridge did not override. Both now fail immediately. A websocket drop
  while starting does too.
- **`UseFrontCamera` is honoured on Android.** It was Apple-only; the Android builder's video
  source is now set from it.
- **Stale callbacks are dropped on Android.** The native client is rebuilt per operation, but
  the old one could still deliver a late callback — a publish that timed out and was retried
  could be "completed" by the abandoned session. Bridges now carry a session token and discard
  what does not belong to the current one.
- `IsStreaming` is set before `PublishStarted`/`PlayStarted` fire, so a handler reading it sees
  the new state; the one-operation-at-a-time guard is now race-free.

### Errors you can program against

`AntMediaErrorEventArgs` and `AntMediaException` gained a **`Code`** — a small
`AntMediaErrorCode` enum (`NoStreamExists`, `StreamIdInUse`, `Unauthorized`, `Timeout`,
`Disconnected`, …) mapped from the server's stable error definitions on both platforms, so
"stream taken, pick another id" no longer means parsing a message whose wording differs per
platform. `AntMediaException` also gained the standard `(string, Exception)` constructor.

### New options

| Option | What it does |
| --- | --- |
| `VideoBitrateKbps` | Bounds video quality against bandwidth. Android applies it as the encoder's starting bitrate, Apple as a ceiling. 0 = SDK default. |
| `AudioBitrateKbps` | Android only — the Apple SDK has no audio bitrate control. |
| `ReconnectionEnabled` | On by default. Android honours `false`; the Apple SDK always reconnects and cannot be told not to. |

Options are also validated now (positive capture dimensions and rates, non-negative bitrates, a
positive timeout — or `Timeout.InfiniteTimeSpan`, which is newly supported), and the docs state
the mutation contract: the client re-reads the options at each start.

### MAUI

- **`AntMediaVideoView` has bindable properties**: `Scaling` (`Fill`/`Fit`/`Balanced`),
  `Mirror` (the selfie-preview flip, on both platforms), and `IsMediaOverlay` — the Android
  `SurfaceView` z-order switch that overlapping-preview layouts silently need. Unset, every
  platform keeps its previous defaults.
- **`SetLocalView`/`SetRemoteView` no longer throw when the view has no handler yet** — the
  attachment defers until the handler is created, so calling them from a page constructor works.
  (On Android, still make sure the view is on screen before publishing: the SDK takes the
  renderer at session start.)
- **The Android `Activity` is resolved per session, not captured forever.** `CreateClient()`
  hands the client a provider, so a client that lives across a configuration change builds its
  next session against the current activity instead of a destroyed one. Plain-Android users get
  the same via a new `AntMediaClient(options, Func<Activity>)` constructor.
- A stable XAML namespace: `xmlns:antmedia="http://schemas.sbokatuk.com/antmedia/maui"` (the
  `clr-namespace:` form still works).

### Bindings

- iOS/Mac: `SetMicMute` gained an awaitable `SetMicMuteAsync` counterpart, and the hand-written
  binding surface is now fully documented, so IntelliSense has something to say on every facade
  member.
- The sample handles server-side session ends (buttons re-enable on disconnect), rebuilds its
  client when the server url is corrected, and demonstrates `Scaling`/`Mirror` and the new
  error codes.

### Packaging and supply chain

- **Every package now ships `THIRD-PARTY-NOTICES.txt`** with the licence texts for the
  redistributed Ant Media SDKs (MIT) and libwebrtc (BSD) — the BSD notice requirement was not
  being met by the MIT `PackageLicenseExpression` alone.
- **The native inputs are integrity-pinned.** The Android tag — tags are mutable — is anchored
  to its commit sha and asserted at fetch; the Mac Catalyst libwebrtc download is verified
  against a pinned sha256; and a new weekly workflow watches both upstreams for movement and
  re-verifies the pinned release asset's bytes, opening an issue on drift.
- **Releases carry GitHub build provenance attestations**
  (`gh attestation verify <package> --repo sbokatuk/AntMedia.Net`), and PR betas are no longer
  published on every push — publishing needs the `publish-beta` label.
- CI now compiles the `net8.0-*` legs of the packages (previously only structurally checked),
  and the package tests additionally pin the Android GoogleGson dependency groups, the symbol
  packages' per-framework pdbs, and the notices file.
- The nuget.org package pages no longer lead with dead links — the shared README uses absolute
  urls, and its quick start now actually starts: `UseAntMedia()`, the XAML namespace, and the
  create-after-appearing rule are all in it.

## Upgrading

Three deliberate behaviour changes to check:

- **`SendData(byte[])` now defaults to `binary: true`.** Raw bytes were being flagged as text,
  which made peers UTF-8-decode them. If you relied on the old default for text, use
  `SendData(string)` or pass `binary: false` explicitly.
- **`Stop()` during a pending start throws `OperationCanceledException` in the awaiter** (and
  `Dispose()` an `ObjectDisposedException`) where the old behaviour was an `AntMediaException`
  after the timeout expired. Catch `OperationCanceledException` if you stop while starting.
- **Options are validated.** A zero timeout or a negative capture size now throws an
  `InvalidOperationException` at the next `PublishAsync`/`PlayAsync` instead of failing in the
  SDK in some platform-specific way.
